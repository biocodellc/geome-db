Genomic Observatories MetaDatabase (GeOMe)

The Genomic Observatories Meta-Database (GeOMe) is a web-based database which captures metadata on biological samples, used for biodiversity inventories, population studies, and environmental metagenomics. GeOMe assigns persistent identifiers for all samples and sampling events and specifies the set of metadata attributes which satisfy the requirements of the genomic observatories model, including capturing the who, what, where, and when associated with all samples. GeOMe provides instant feedback to users on the quality of their data and packages data for further analysis for use in a laboratory information system (LIMS) using the Biocode LIMS plugin. GeOMe also packages submissions for easy delivery to the Sequence Read Archive (SRA) and Genbank's Nucleotide database.

GeOMe is supported by a grant from the National Science Foundation (DEB 1457848)

Users can upload two types of data (under "tools" tab)

To upload data users much have an account. If you don't have one please send an email with you name, title and affiliation to geome.database@gmail.com

  1. Published Sanger sequence data and the associated metadata (fasta format)
  2. Metadata associated with next-generation sequence data (fastq format)
    • In the process of uploading the metadata for your next-generation sequence data we create two files:
      1. the SRA metadata file
      2. the BioSample datasheet
       

This will make uploading your data to Genbank's Short Read Archive (SRA) database faster and easier.

* Note we are not currently accepting microsatellite datasets

What you need to do before you upload data

  • Generate a template (under “tools” tab) for your metadata
    • This option allows the user to create an excel file to store the metadata associated with each of their samples.
